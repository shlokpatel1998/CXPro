## Codebase Patterns
- Use bp-* CSS class names for all UI elements — no raw Tailwind color classes
- Supabase relational joins only work through actual FKs — use multi-step queries + JS join when tables share user_id but have no direct FK
- Theme state lives in layout.tsx (localStorage + data-theme on documentElement)
- NEXT_PUBLIC_API_URL env var for all backend fetch calls (fallback: http://localhost:8000)
- Auth check pattern: supabase.auth.getSession() → if no session redirect to /auth
- getErrorMessage from @/lib/error for all error display — no instanceof Error pattern
- maybeSingle() for queries that may legitimately return zero rows; single() only when row must exist
